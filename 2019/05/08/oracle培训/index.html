Oracle培训记录

发表于 2019-05-08 | 分类于 培训记录 | 评论数： | 阅读次数： | 阅读次数：
本文字数： 1.1k | 阅读时长 ≈ 1 分钟

2019年5月8号至5月9号两天有幸参加了甲骨文杭州的培训,针对本次培训,通过本文进行记录。

两天培训

第一天

  • ADW介绍以及使用(主要是数据导入)
  • ADW服务于SasS PaaS IaaS
  • Exadata + oracle database + oracle cloud 实现”自治数据库”(包括:ADW(国内主推) or ATP
  • OAC介绍以及使用
  • OAC学习网站
  • oracle文档网站

第二天

十个小实验

实验一

  • 创建oracle自治仓库云(ADWC)(即Autonomous Data Warehouse Cloud)实例,通过sql developer桌面客户端连接ADWC(通过console生成的wallet进行安全连接):
    1. 登录oracle cloud控制台,选择Autonomous Data Warehouse,按照流程创建ADW
    2. 点击上一步创建的ADWC实例,选择”Service console”;在新页面上点击”Administration”,点击”Download Client Credentials”下载wallet文件(不需要解压)
    3. 打开sql developer桌面版,连接创建的ADWC(需选择wallet文件进行安全验证)

实验二

  • 比较ADWC三种数据模式各个应用场景和效率:

    • medium
    • low
    • high

  • 具体区别见图:

实验三

  • ADWC数据加载,加载数据至ADWC:
    • 从本地文件导入数据
    • 从OCI对象存储导入数据
      1. 上传文件至OCI对象存储中
      2. 使用 PL / SQL 包 DBMS_CLOUD 加载数据,进行故障排除

实验四

  • 外部数据与ADWC相关:
    • 查询外部数据(从oracle云对象存储中)
    • 从外部数据导入表(和lab3不一样的是,lab3需要提前创建表结构,这里是根据外部数据自行设置外部表,创建表结构,导入数据)

实验五

  • 创建OAC(Oracle Analytics Cloud):
    1. 进入我的仪表盘
    2. 选择”分析”,创建一个OAC实例
    3. OAC有三个版本，每个版本各自有不同特性。

实验六

  • 将ADW与OAC连接,将ADW数据导入到OAC中进行数据分析,在OAC面板中:
    1. 创建连接(与ADW连接)
    2. 创建数据集(从ADW选择需要用到的数据构建数据集)
    3. 修改数据类型(属性/度量)让数据在不同维度下展示
    4. 选择合适的数据集构建项目
    5. 按照需求构建图表进行分析服务业务
    6. 导出为分享链接/本地dvd文件通过桌面dvd软件进行本地的图标构建数据分析
